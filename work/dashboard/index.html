Workflow Dashboard
A self-built tool designed to organize projects, tasks and next steps in one place.
Product Thinking Information Architecture Workflow Design Self-Initiated
Self-initiated · 2026 · Next.js · Built with Claude Code
Instead of adapting my workflow to another tool, I built my own.
Challenge
Applications, notes, documents and project information were spread across too many tools.

I wanted a system that matched the way I actually work. Instead of adapting to another tool, I started building my own.

Live Demo

See it in action.

A Studio version of the dashboard — the same structure used to organize creative project tasks. Kanban columns, status tracking, detail panels. Fully interactive.

vivien.studio — dashboard
Interactive Demo

Click any card to open the detail panel. The structure — kanban columns, status tracking, project filters — is the same across all use cases.

Process
How I approached it.
  • 01 Identified recurring workflow problems — too many tools, no overview, constant context-switching
  • 02 Structured information into clear categories before writing a single line of code
  • 03 Built a working prototype fast — the goal was something real and usable, not perfect
  • 04 Improved the system through daily use — every change was based on where it broke down in practice
What I learned

"I don't just enjoy designing how things look —
I enjoy designing how things work."

Building the dashboard strengthened my interest in product thinking, information architecture and digital tools.
